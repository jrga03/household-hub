# Source Code (`/src/`)

## Purpose

This directory contains all application source code for Household Hub, implementing an offline-first Progressive Web App with event sourcing and multi-device sync. The architecture follows a three-layer state pattern: **Zustand (UI) → IndexedDB (persistent) → Supabase (cloud truth)**.

## Architecture Layers

```
┌─────────────────────────────────────────────────┐
│  UI Layer (React + TanStack Router)            │
│  routes/ + components/                          │
└─────────────────┬───────────────────────────────┘
                  │
┌─────────────────▼───────────────────────────────┐
│  Data Layer (Hooks + Stores)                   │
│  hooks/ (TanStack Query + custom)              │
│  stores/ (Zustand client state)                │
└─────────────────┬───────────────────────────────┘
                  │
┌─────────────────▼───────────────────────────────┐
│  Business Logic Layer                          │
│  lib/ (utilities, sync, offline, events)      │
└─────────────────┬───────────────────────────────┘
                  │
┌─────────────────▼───────────────────────────────┐
│  Storage Layer                                 │
│  IndexedDB (Dexie) ←→ Supabase                │
└─────────────────────────────────────────────────┘
```

## Contents

### Entry Points

- **`main.tsx`** - Application entry point
  - Initializes Sentry error tracking
  - Sets up TanStack Query client
  - Wraps app with AuthProvider and QueryClientProvider

- **`App.tsx`** - Root React component (main.tsx:4)
  - Creates TanStack Router instance from `routeTree.gen.ts`
  - Initializes realtime sync and background sync
  - Sets up event compaction (daily at 3 AM)
  - Renders global UI (OfflineBanner, SyncIndicator, PWA prompts)

- **`sw.ts`** - Service worker for PWA functionality (sw.ts:1)
  - Workbox-based caching strategies
  - Push notification handlers
  - Offline asset precaching

### Configuration Files

- **`index.css`** - Global styles and Tailwind CSS imports
- **`vite-env.d.ts`** - Vite TypeScript declarations
- **`routeTree.gen.ts`** - Auto-generated TanStack Router route tree (DO NOT EDIT)
  - Regenerated automatically when routes change
  - Provides type-safe routing

### Core Directories

- **[`components/`](./components/)** - All React UI components
  - Feature-based organization (budgets/, analytics/, charts/, etc.)
  - Standalone components for shared functionality
  - shadcn/ui component library in `ui/` subdirectory

- **[`hooks/`](./hooks/)** - Custom React hooks
  - Data fetching hooks (TanStack Query)
  - Offline data hooks (IndexedDB)
  - PWA hooks (install prompt, service worker, storage)
  - Utility hooks (keyboard shortcuts, media queries)

- **[`lib/`](./lib/)** - Core business logic and utilities
  - Sync engine (event sourcing, conflict resolution)
  - Offline operations (sync queue, transactions)
  - Database layer (Dexie.js)
  - Currency utilities, validation, import/export

- **[`routes/`](./routes/)** - TanStack Router page components
  - File-based routing convention
  - Protected routes (authentication required)

- **[`stores/`](./stores/)** - Zustand client state stores
  - Lightweight UI state (not server data)
  - Auth store, sync status, navigation state

- **[`types/`](./types/)** - TypeScript type definitions
  - Entity types (accounts, transactions, categories)
  - Database types (auto-generated from Supabase)
  - Sync and event types

- **`test/`** - Test configuration
  - Vitest setup file

## Key Concepts

### Import Path Aliases

All source files use the `@/` alias that resolves to the `src/` directory (configured in `vite.config.ts` and `tsconfig.json`):

- `@/lib/currency` → `src/lib/currency.ts`
- `@/components/ui/button` → `src/components/ui/button.tsx`
- `@/stores/authStore` → `src/stores/authStore.ts`

### State Management Strategy

**Server State (TanStack Query):**

- Accounts, transactions, categories, budgets from Supabase
- Automatic caching, refetching, and background updates
- See `hooks/` for query hooks

**Client State (Zustand):**

- Authentication state, UI toggles, navigation
- Lightweight, no persistence (except auth)
- See `stores/` for available stores

**Offline State (IndexedDB via Dexie):**

- Persistent local storage for offline-first
- Sync queue for pending changes
- See `lib/dexie/` and `lib/offline/`

### Auto-Generated Files

The following files are **auto-generated** and should not be edited manually:

- `routeTree.gen.ts` - Generated by TanStack Router when routes change
- `types/database.types.ts` - Generated from Supabase schema (`npx supabase gen types`)

## Common Development Tasks

### Adding a New Page/Route

1. Create file in `routes/` (e.g., `reports.tsx`)
2. TanStack Router auto-generates route in `routeTree.gen.ts`
3. Add navigation link in `components/layout/AppSidebar.tsx`
4. See [routes/README.md](./routes/) for routing conventions

### Creating a New Component

1. Add file to `components/` or feature subdirectory
2. Import from routes or other components: `import { MyComponent } from "@/components/MyComponent"`
3. Use shadcn/ui components from `components/ui/`
4. See [components/README.md](./components/) for organization patterns

### Adding a New Custom Hook

1. Create file in `hooks/` (e.g., `useFeature.ts`)
2. Prefix with `useOffline` if it reads from IndexedDB
3. Use TanStack Query for server data fetching
4. See [hooks/README.md](./hooks/) for hook patterns

### Working with Currency

All currency operations use PHP (Philippine Peso) stored as **integer cents**. Use `formatPHP()` to display amounts and `parsePHP()` to convert user input to cents. See `lib/currency.ts` and `lib/currency.md` for complete documentation.

### Debugging Sync Issues

1. Check `SyncIndicator` component (top-right of app)
2. Open browser console for sync logs
3. Inspect `sync_queue` table in IndexedDB (DevTools → Application)
4. See [lib/sync/README.md](./lib/sync/) and [lib/offline/README.md](./lib/offline/) for sync architecture

## Testing

### Unit Tests (Vitest)

Tests are colocated with source files:

```bash
npm test                    # Run all unit tests
npm test -- currency        # Run specific test file
npm test -- --coverage      # With coverage report
```

Test files: `*.test.ts` or `*.test.tsx` (e.g., `lib/currency.test.ts`)

### E2E Tests (Playwright)

Located in `/tests/e2e/`:

```bash
npm run test:e2e           # Run all E2E tests
npm run test:e2e:ui        # Run with Playwright UI
```
