# Static Assets (`/public/`)

## Purpose

The public directory contains **static assets** served directly by the web server without processing. These files are available at the root URL path.

## Contents

- **`offline.html`** - Offline fallback page (service worker)
- **`icons/`** - PWA icon set (various sizes)

## offline.html

**Purpose:** Displayed when user is offline and navigates to uncached route

**Accessed by:** Service worker (sw.ts)

**Content:**

- Simple message: "You are offline"
- Instructions to return online
- Styled to match app theme

**Customization:**

```html
<!DOCTYPE html>
<html>
  <head>
    <title>Offline - Household Hub</title>
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <style>
      /* Inline styles (no external CSS) */
    </style>
  </head>
  <body>
    <h1>You're offline</h1>
    <p>Please check your internet connection.</p>
  </body>
</html>
```

## icons/

**Purpose:** PWA icons for various devices and contexts

**Generated by:** `scripts/generate-icons.js`

**Icon Sizes:**

- `icon-192x192.png` - Android Chrome, PWA install
- `icon-512x512.png` - Android Chrome splash screen
- `apple-touch-icon.png` - iOS home screen icon

**Referenced in:** `index.html` and PWA manifest

**Updating Icons:**

1. Update source icon design
2. Run `node scripts/generate-icons.js`
3. Verify new icons generated
4. Test PWA install on devices

## Adding Static Assets

### General Rules

**DO place in public/ when:**

- ✅ Files served as-is (no transformation)
- ✅ Files accessed via root URL (`/file.ext`)
- ✅ Files needed by service worker
- ✅ Static images, fonts, data files

**DON'T place in public/ when:**

- ❌ Files imported in components (use `src/assets/`)
- ❌ Files need processing (optimization, bundling)
- ❌ Files change frequently during development

### Example: Adding Robot.txt

**1. Create file:**

```
# public/robots.txt
User-agent: *
Allow: /

Sitemap: https://household-hub.com/sitemap.xml
```

**2. Access at:** `https://household-hub.com/robots.txt`

### Example: Adding Favicon

**1. Add file:**

```
public/favicon.ico
```

**2. Reference in index.html:**

```html
<link rel="icon" href="/favicon.ico" />
```

## File Access

**Public files are served at root:**

- `public/offline.html` → `https://example.com/offline.html`
- `public/icons/icon-192x192.png` → `https://example.com/icons/icon-192x192.png`
- `public/robots.txt` → `https://example.com/robots.txt`

**⚠️ Do NOT prefix with `/public/` in URLs**

```html
<!-- ✅ Correct -->
<img src="/icons/icon-192x192.png" />

<!-- ❌ Wrong -->
<img src="/public/icons/icon-192x192.png" />
```

## Service Worker Integration

**Service worker accesses public files:**

```javascript
// sw.ts
precacheAndRoute([
  { url: "/offline.html", revision: "1" },
  { url: "/icons/icon-192x192.png", revision: "1" },
]);

// Offline fallback
registerRoute(
  ({ request }) => request.mode === "navigate",
  async () => {
    try {
      return await fetch(request);
    } catch {
      return caches.match("/offline.html");
    }
  }
);
```

## Performance Considerations

**Optimize assets:**

- Compress images (use WebP when possible)
- Minify HTML/CSS in static pages
- Use appropriate caching headers

**Small files only:**

- Large files should be in CDN or cloud storage
- Public folder increases deployment size
